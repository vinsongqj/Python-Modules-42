# Python-Modules-42

## Description

In Milestone 2 of the new Common Core of 42, we were tasked with completing 11 python modules, each with a different learning outcomes:

**Module 00** 
- Function definition
- Basic I/O
- Data type casting
- if/else
- range(),
- Recursion
- Type hinting
  
**Module 01**
- Classes and instances
- Constructors (``__init__``)
- Encapsulation
- Getter/setters
- Inheritance with ``super()``
- ``@staticmethod`` and ``@classmethod``
- Nested classes
- Shebang ``#!``
  
**Module 02**
- ``try/except`` blocks
- Raising errors
- Distinguishing errors (``ValueError``, ``ZeroDivisionError``, ``FileNotFoundError``, ``TypeError``)
- Custom ``Exception`` classes and hierarchies
- ``finally`` blocks
- Input sanitation
  
**Module 03**
- Lists
- Tuples
- Sets
- Dictionaries
- Comprehensions
- Generators, iterators and ``yield``
  
**Module 04**
- Reading files with ``open()`` and ``read()``
- Writing to files with ``write()``
- Closing files with ``close()``
- ``sys.stderr`` and ``sys.stdin``
- Context managers with ``with()``
  
**Module 05**
- Polymorphism
- ``abc`` module and ``@abstractmethod`` decorator usage
- Method overriding
- Stream architecture
  
**Module 06**
- Initializing packages with ``__init__.py``
- Package structuring
- Absolute imports
- Relative imports
- Circular dependencies
  
**Module 07**
- Decoupling instantiation
- Encapsulation
- Interface segregation
- Behavioral decoupling
  
**Module 08**
- Virtual environments
- Poetry and pip
- ``requirements.txt`` and ``pyproject.toml``
- ``.gitignore``
  
**Module 09**
- Pydantic type coercion and type enforcement
- ``Field``
- Alias mapping
- ``@field_validator`` and ``@model_validator``
- Nested hiearchies
  
**Module 10**
- Higher order functions
- Lambdas
- Using functools (``map()``, ``filter()``, ``reduce()``)
- Decorators
- Lexical scoping

*Please note that some of these modules are outdated as many changes have been made to the project requirements since their completion.*
